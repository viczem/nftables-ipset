# nftables-ipset – Simple IP blocklist manager

A tiny command‑line tool to keep a **persistent IP blocklist** in a SQLite database and to generate a ready‑to‑use *nftables* set file.
It is deliberately minimal: one Python script, no external services, and everything is stored locally.

Typical workflow:

1. **Add** single or many IP addresses (optionally with metadata).
2. **Remove** single or many IP addresses.
3. **Export** the whole list to a nftables‑compatible file (`20‑blocklist‑ipv4.nft`).

That file can be dropped into `/etc/nftables.d/` and automatically included by the global nftables configuration.

## Configuration

### Database location

| Variable | Default | Example |
|----------|---------|---------|
| `DIR`    | *directory where the script resides* | `export DIR=/var/lib/nftables-ipset` |

- The directory must be **writable** by the user that runs the script (or by the systemd service, if you use it).
- The database file is always named `nftables-ipset.db`.

### nftables file layout (the “canonical” layout used by the author)

```plain
/etc/nftables.d/
├── 10-blocklist.nft          # static table / chain definition (never overwritten)
└── 20-blocklist-ipv4.nft    # generated by `ipset --export`
```

**10‑blocklist.nft** (provided with the README, copy it to the host)

```nft
table inet blocklists {
    set blocklist_ipv4 {
        type ipv4_addr
        flags interval
    }

    chain earlydrop {
        type filter hook prerouting priority -300;
        policy accept;

        ip saddr @blocklist_ipv4 drop
    }
}
```

**20‑blocklist‑ipv4.nft** (generated)

```nft
add element inet blocklists blocklist_ipv4 {
    1.1.1.1,
    2.2.2.2,
    …
}
```

**/etc/nftables.conf** (the master file, also provided by most distros)

```bash
#!/usr/sbin/nft -f

flush ruleset
#...
include "/etc/nftables.d/*.nft"
```

The `include` line automatically picks up both `10‑blocklist.nft` and `20‑blocklist‑ipv4.nft`.
When you run `nftables-ipset --export`, the generated file is **overwritten** in place, so the next `nft -f /etc/nftables.conf` reload picks up the new list.

## Usage

```bash
uv run nftables-ipset -h
```

```plain
usage: nftables-ipset [-h] [-a IP] [-A] [-r IP] [-R] [-m METADATA] [-e]

Manage an IP blocklist stored in a SQLite database. The location of the
database directory can be overridden with the environment variable DIR
(default: the script's directory).

options:
  -h, --help            show this help message and exit
  -a IP, --add IP       Add a single IP address (metadata optional via -m).
  -A, --batch-add       Add many IPs from stdin (empty line / Ctrl‑D ends).
  -r IP, --remove IP    Remove a single IP address.
  -R, --batch-remove    Read many IPs from stdin (empty line / Ctrl‑D ends) and delete them.
  -m METADATA, --metadata METADATA
                        If supplied, this string will be stored as the metadata
                        for EVERY IP added (overrides per‑line comments).
  -e, --export          Export all stored IPs to 20-blocklist-ipv4.nft (nftables format).
```

### Adding a single address

```bash
uv run nftables-ipset --add 203.0.113.45
# with metadata (e.g., source of the block)
uv run nftables-ipset --add 203.0.113.45 --metadata "spam‑source:spamhaus"
```

### Adding many addresses (batch mode)

```bash
# From a file
cat bad_ips.txt | uv run nftables-ipset --batch-add

# From another command
curl -s https://example.com/blocked.txt | uv run nftables-ipset --batch-add
```

*Batch mode reads one IP per line. An empty line or `Ctrl‑D` ends the input.*

If the input file contains **comments** (anything after a `#`), they are stripped automatically.
If you want **the same metadata** for every line, pass `-m "your‑note"`; it overrides per‑line comments.

### Removing a single address

```bash
uv run nftables-ipset --remove 203.0.113.45
```

### Removing many addresses (batch mode)

```bash
cat to_remove.txt | uv run nftables-ipset --batch-remove
```

### Exporting to nftables

```bash
uv run nftables-ipset --export
```

The command:

1. Reads **all rows** from the database.
2. Writes them into `20-blocklist-ipv4.nft` **in the same directory as the script** (or `DIR` if set).
3. Copies the file to `/etc/nftables.d/20-blocklist-ipv4.nft` (requires root permissions).

After export you typically reload nftables:

```bash
sudo nft -f /etc/nftables.conf
# or, if you have a systemd service:
sudo systemctl reload nftables
```
